**版本**：v1.0

**適用範圍**：新增德文（de）做為 UI 在地化、新增簡體中文（zh-CN）與韓文（ko）做為目標學習語言

## 類別一、基礎建設

### 1.1 新建 `src/utils/languageMap.ts`

**目的**：將所有語言相關的對照資訊集中管理，消除散落在各檔案中的硬編碼語言判斷。

**實作內容**：

1. 定義 `LanguageInfo` 類型介面，包含：
   - `langCode`：標準 BCP 47 語言標籤（如 `'de-DE'`）
   - `ttsLangCode`：TTS 引擎使用的 BCP 47 標籤
   - `ttsPrefix`：TTS 前綴（如 `'de'`）
   - `whisperLang`：Whisper API 的語言參數
   - `isTargetLanguage`：是否為目標學習語言
   - `isUiLanguage`：是否為 UI 介面語言
   - `uiLabel`：UI 選單中顯示的名稱（多語言版本）
   - `targetLabel`：目標學習語言名稱（依 UI 語言動態顯示）

2. 建立 `LANGUAGE_MAP` 常數物件，金鑰為語言代碼，值為上述資訊：
   - `en`：英文（現有，目標學習語言）
   - `ja`：日文（現有，目標學習語言）
   - `zh-CN`：簡體中文（新增，目標學習語言）
   - `ko`：韓文（新增，目標學習語言）
   - `zh-TW`：繁體中文（現有，UI 語言）
   - `en`：英文（現有，UI 語言）
   - `ja`：日文（現有，UI 語言）
   - `ko`：韓文（現有，UI 語言）
   - `es`：西班牙文（現有，UI 語言）
   - `fr`：法文（現有，UI 語言）
   - `de`：德文（新增，UI 語言）

3. 提供工具函數：
   - `getLangInfo(langId)`：取得指定語言的完整資訊
   - `getTtsCode(langId)`：取得 TTS 語言標籤
   - `getWhisperLang(langId)`：取得 Whisper 語言參數
   - `getLangName(langId, uiLang?)`：取得語言顯示名稱
   - `getTargetLanguages()`：取得所有目標學習語言清單
   - `getUiLanguages()`：取得所有 UI 語言清單
   - `isTarget(langId)`：判斷是否為目標學習語言
   - `isUiLang(langId)`：判斷是否為 UI 語言

4. **現有散落的硬編碼位置（需在完成後取代）**：
   - `src/prompts/analyze.ts` 第 5-6 行：`langName`、`termName` 判斷
   - `src/prompts/chat.ts` 第 5 行：`langName` 判斷
   - `src/prompts/polish.ts` 第 5 行：`langName` 判斷
   - `src/prompts/conversation.ts` 第 5 行：`langName` 判斷
   - `src/utils/llmClient.ts` 第 133、199-201、229 行：`langName`、Whisper 參數
   - `src/components/Chat.tsx` 第 335、402、431、451、453 行：`isJa` 判斷、TTS 參數
   - `src/components/Curriculum.tsx` 第 17-18 行：`isJa` 判斷
   - `src/components/Settings.tsx` 第 61-64 行：`targetLangOptions` 陣列

## 類別二、提示詞與 API

**目的**：讓 LLM 提示詞能根據目標語言動態調整，不再寫死「英文／日文」。

### 2.1 修改 `src/prompts/analyze.ts`

**現況**：第 5-6 行硬編碼 `langName` 與 `termName`；第 27 行硬編碼 `vocabFormatInstruction`

**修改方式**：

- 第 5 行：移除 `ctx.targetLanguage === 'en' ? '美語' : '日語'` 的三元運算，改為呼叫 `getLangName(ctx.targetLanguage, ctx.uiLang)`
- 第 6 行：移除 `ctx.targetLanguage === 'en' ? '英文單字或片語' : '日文單字、假名或片語'` 的三元運算，改為呼叫 `getTermName(ctx.targetLanguage, ctx.uiLang)`
- 第 27 行：移除 `vocabFormatInstruction` 中的 `targetLanguage === 'en'` 判斷，改為通用邏輯（例如：英文用「音標」，日文用「假名」）

### 2.2 修改 `src/prompts/chat.ts`

**現況**：第 5 行硬編碼 `langName`；第 43 行硬編碼 `vocabFormatInstruction`

**修改方式**：同 2.1，使用 `getLangName()` 與通用邏輯。

### 2.3 修改 `src/prompts/polish.ts`

**現況**：第 5 行硬編碼 `langName`

**修改方式**：改為呼叫 `getLangName()`。

### 2.4 修改 `src/prompts/conversation.ts`

**現況**：第 5 行硬編碼 `langName`

**修改方式**：改為呼叫 `getLangName()`。

### 2.5 修改 `src/utils/llmClient.ts`

**現況**：

- `transcribeAudio` 函數（第 191 行起）：僅處理 `en` 與 `ja` 的 Whisper 語言參數
- `polishSentenceAPI` 函數（第 133 行）：硬編碼 `langName`

**修改方式**：

- `transcribeAudio`：將 `if (targetLanguage === 'en')` / `else if (targetLanguage === 'ja')` 改為查表 `getWhisperLang(targetLanguage)`
- `polishSentenceAPI`：將 `const langName = targetLanguage === 'en' ? '英文' : '日文'` 改為呼叫 `getLangName(targetLanguage, uiLang)`
- 其他函數若有類似硬編碼，一併修正

## 類別三、前端元件

**目的**：讓語音合成、UI 文字能根據目標語言與 UI 語言動態調整。

### 3.1 修改 `src/components/Chat.tsx`

**現況**：

- 第 335 行：`const isJa = targetLanguage === 'ja';`
- 第 402、431 行：`if (!isJa)` 判斷
- 第 451 行：`const targetLangCode = isJa ? 'ja-JP' : (targetLanguage === 'en' ? 'en-US' : getLangCode(targetLanguage));`
- 第 453 行：`const targetLangPrefix = isJa ? 'ja' : (targetLanguage === 'en' ? 'en' : getLangPrefix(targetLanguage));`

**修改方式**：

- 移除 `isJa` 變數，改為使用 `getLangInfo(targetLanguage)` 取得的 TTS 參數
- 將所有 `isJa` 判斷改為 `langInfo.ttsLangCode` / `langInfo.ttsPrefix` 的通用查表
- TTS 段落處理邏輯保持不變，但使用動態取得的語言參數

### 3.2 修改 `src/components/ChatWrapper.tsx`

**現況**：多處 `targetLanguage === 'en'` 判斷，用於顯示 AI 問候語、翻譯提示等

**修改方式**：

- 將所有 `targetLanguage === 'en' ? '英文' : '日文'` 之類的硬編碼，改為呼叫 `getLangName(targetLanguage, uiLang)`
- 動態產生 AI 問候語與翻譯提示文字

### 3.3 修改 `src/components/Curriculum.tsx`

**現況**：第 17-18 行硬編碼 `isJa` 與 `langCode`；第 27、29 行硬編碼 TTS 參數

**修改方式**：

- 移除 `isJa` 變數，改為使用 `getLangInfo(lang)` 取得的資訊
- TTS `utterance.lang` 改為 `getTtsCode(lang)`
- 前綴改為 `getLangInfo(lang).ttsPrefix`

### 3.4 修改 `src/components/Dashboard.tsx`

**現況**：第 51 行顯示「英語／日語」，硬編碼

**修改方式**：

- 改為呼叫 `getLangName(targetLanguage, uiLang)` 動態顯示

### 3.5 修改 `src/components/Settings.tsx`

**現況**：第 61-64 行 `targetLangOptions` 僅有英文與日文選項

**修改方式**：

- `targetLangOptions` 改為呼叫 `getTargetLanguages()` 動態產生
- 或手動新增 `zh-CN` 與 `ko` 選項（若 `languageMap` 尚未完成）
- 若有 `uiLang` 選單，加入德文（`de`）選項

## 類別四、課程與句型資料

**目的**：為新增的目標學習語言（簡體中文、韓文）建立對應的課程結構與句型練習資料。

### 4.1 擴充 `src/data/curriculumData.ts`

**現況**：現有 `ja: [...]` 與 `en: [...]` 陣列，尚無 `zh-CN` 與 `ko`

**修改方式**：

1. 新增 `zh-CN: [...]` 陣列：
   - 結構與現有 `ja` / `en` 相同
   - 每個單元的 `title` 需包含多語言版本（`zh-TW`、`zh-CN`、`en`、`ja`、`ko`、`es`、`fr`、`de`）
   - `description`、`objectives`、`activities` 等欄位同樣多語言化
   - 建議初版先複製 `en` 的單元結構，翻譯標題與說明即可

2. 新增 `ko: [...]` 陣列：
   - 同上，複製 `ja` 的單元結構（因日文與韓文學習模式較接近），翻譯為韓文標題與說明

3. 確認現有 `en` 與 `ja` 陣列中的 `title` 已有 `de` 欄位（若沒有，需補上）

### 4.2 新增 `src/data/scenarioPatterns/zh-CN/` 目錄與檔案

**目的**：為簡體中文學習者提供句型練習資料

**修改方式**：

1. 在 `src/data/scenarioPatterns/` 下建立 `zh-CN/` 目錄
2. 參考 `en/` 或 `ja/` 的 `01.json`、`02.json` 結構，建立對應的簡體中文版本
3. JSON 內容需包含：
   - `version`：版本號
   - `patterns`：句型陣列，每個句型包含：
     - `pattern`：句型主體
     - `explanation`：說明（多語言版本）
     - `examples`：範例陣列
     - `substitutionSlots`：可替換位置
     - `difficulty`：難度等级
     - `tags`：標籤

### 4.3 新增 `src/data/scenarioPatterns/ko/` 目錄與檔案

**目的**：為韓文學習者提供句型練習資料

**修改方式**：

1. 在 `src/data/scenarioPatterns/` 下建立 `ko/` 目錄
2. 同 4.2，建立韓文版本的句型練習資料
3. 可參考 `ja/` 的結構，因日文與韓文語法有一定相似性

### 4.4 確認 `src/data/scenarioPatterns/index.js`

**現況**：動態載入 `/data/scenarioPatterns/${langId}/${version}.json`

**確認事項**：

- 確保路徑解析正確，支援新建立的 `zh-CN/` 與 `ko/` 目錄
- 若有問題，檢查 Vite 公開資料夾設定（`public/data/scenarioPatterns/`）

## 類別五、德文 UI 在地化

**目的**：讓 App 的 UI 介面可以切換成德語顯示，擴大德語使用者群。

### 5.1 新增 `src/i18n/locales/de.json`

**目的**：翻譯所有 UI 字串為德文

**修改方式**：

1. 在 `src/i18n/locales/` 下建立 `de.json`
2. 參考現有語言檔（`zh-TW.json`、`en.json` 等）的結構，翻譯所有 key 為德文
3. 預估翻譯數量：約 757 條 key（與其他語言一致）
4. 翻譯原則：
   - 標點符號使用德文慣例（句號 `.`、逗號 `,`）
   - 人稱代詞大小寫（德文規則）
   - 專業術語統一（如「文法」→ `Grammatik」）
   - 確認測試階段需人工校對

### 5.2 更新 `src/i18n/index.ts`

**現況**：管理所有語言檔的引入與註冊

**修改方式**：

1. 引入 `de.json`
2. 在 `uiLanguages` 或類似陣列中加入 `de`
3. 確保載入順序正確（德文擺放位置不影響功能）

### 5.3 更新 `src/components/Settings.tsx` 的 UI 語言選單

**現況**：若有 `uiLang` 選單，可能尚未包含德文選項

**修改方式**：

- 在 UI 語言選單中加入德文（`de`）選項
- 使用 `t('德文 (Deutsch)')` 或從 `languageMap` 動態產生

### 5.4 確認 TTS 對德語 UI 的支援（選用）

**目的**：若希望德語 UI 也能有語音朗讀（例如按鈕提示）

**確認事項**：

- 檢查 `Chat.tsx` 的 `handleSpeak` 函數是否支援德語（`de`）TTS
- 若不支援，加入 `de` 的 TTS 語言參數至 `languageMap`

## 實作順序建議

1. **基礎建設**：`languageMap.ts`（類別一）
   - 所有後續修改都依賴此檔案，先完成此檔可避免到處修改參數

2. **提示詞與 API**：類別二
   - 提示詞影響 AI 回覆品質，需第一優先處理

3. **前端元件**：類別三
   - UI 層的修改通常最直觀，可在此階段一併修正 UI 顯示

4. **課程資料**：`curriculumData.ts` 擴充（類別四前半）
   - 課程結構變動較少，可與前端並行處理
   - `scenarioPatterns` 的 JSON 製作較費時，可放在最後或委外處理

5. **德文 UI 在地化**：類別五
   - 翻譯工作獨立，可委外或使用翻譯工具輔助

6. **測試與驗證**：類別六
   - 最後階段，確保所有修改正確運作
